💡 NOVEL THERAPIES — Low-Cost AI + Wearable

Approaches that combine on-device AI inference, smartwatch biosensors, and open-source models to provide continuous monitoring, early detection, and therapeutic support — at minimal cost, accessible anywhere.

These are engineering proposals integrating existing sensor hardware (Apple Watch, Galaxy Watch, any HR/HRV device) with on-device ML. They complement — never replace — clinical oncology. Each therapy lists its evidence basis, implementation stack, and estimated cost.

§1 — Continuous Catecholamine Proxy Monitoring

COST: $0 software · existing smartwatch

Problem

Neuroblastoma produces catecholamines (adrenaline, noradrenaline, dopamine). Current detection requires lab urinalysis (VMA/HVA). Tests are periodic, invasive for children, and unavailable in resource-limited settings.

Approach

Catecholamine surges directly affect heart rate, HRV, skin conductance, and skin temperature — all continuously measurable by a smartwatch. A local ML model learns the child's personal baseline and flags anomalous autonomic patterns consistent with catecholamine spikes.

UDT:CatecholamineProxy {
  inputs: {
    hr:          watch.heartRate           // continuous
    hrv:         watch.hrvSDNN             // 5-min windows
    hr_recovery: watch.postActivityDelta   // after movement
    temp:        watch.skinTemp            // circadian
    accel:       watch.accelerometer       // activity context
    sleep:       watch.sleepStages         // autonomic during sleep
  }

  model: SmolLM-360M fine-tuned on:
    public HRV↔catecholamine datasets
    + child's own lab VMA/HVA results (when available)
    → predicts: VMA_proxy [0..1], HVA_proxy [0..1]

  alert_logic: {
    if proxy > baseline + 2σ for > 2 hours:
      → flag ELEVATED CATECHOLAMINE
    if proxy > baseline + 3σ:
      → flag URGENT · notify caregiver
    if circadian pattern inverts (night > day):
      → flag CIRCADIAN DISRUPTION
  }

  training: federated on-device
    each watch trains locally on own data
    no patient data leaves the device
    model weights shared (differential privacy)
}

Evidence

  • VMA/HVA urinary markers have ~95% sensitivity (PMC8201085)
  • Catecholamines directly elevate HR, reduce HRV, increase skin conductance
  • Smartwatch HRV correlates with autonomic state (validated in HeartMath, Oura, Whoop studies)

Implementation

WearShield BioModule protocol → add CatecholamineProxyModule. Runs on-device. WebLLM for inference, ONNX-lite for the regression model. Calibrate against 2-3 lab draws, then continuous proxy.

Stack: WearShield + HealthKit/Samsung SDK · WebLLM SmolLM-360M · ONNX Runtime Mobile · ~2MB model
watch on-device continuous non-invasive

§2 — Vagal Biofeedback for Autonomic Rebalancing

COST: $0 software · watch haptics + audio

Problem

Sympathetic dominance promotes tumor growth. Vagal (parasympathetic) tone has documented tumor-suppressive effects (DOI 10.3389/fonc.2020.00744). Children with neuroblastoma live in chronic sympathetic overdrive — the tumor itself produces the stress hormones that feed it.

Approach

Use smartwatch haptics and guided audio to deliver personalized vagal stimulation protocols. The watch monitors HRV in real-time and adapts the breathing pattern, haptic rhythm, and session duration to maximize vagal tone. The local AI learns which patterns work best for each child.

UDT:VagalProtocol {
  modes: [
    { name: "4-7-8 Breath",     inhale: 4, hold: 7, exhale: 8 },
    { name: "Coherence",        inhale: 5, hold: 0, exhale: 5 },
    { name: "Extended Exhale",   inhale: 4, hold: 0, exhale: 8 },
    { name: "Box Breathing",     inhale: 4, hold: 4, exhale: 4, hold2: 4 },
    { name: "Humming (Bee)",     inhale: 4, hold: 0, exhale: 10, hum: true }
  ]

  adaptive_logic: {
    measure: hrv_baseline (30s pre-session)
    for each breath_cycle:
      measure: hrv_delta, respiratory_sinus_arrhythmia
      if hrv_improving: maintain pattern
      if hrv_plateaued: extend exhale by 1s
      if hrv_declining: switch to simpler pattern

    session_score = (hrv_post - hrv_pre) / hrv_pre
    model.learn(pattern, context, score)
  }

  delivery: {
    haptic:  watch vibration on inhale/exhale transitions
    audio:   generated tone (pitch follows target breathing)
    visual:  bloom ring expansion/contraction on watch face
  }

  evidence_tracking: {
    log: hrv_pre, hrv_post, session_duration, pattern_used
    trend: weekly vagal tone trajectory
    correlate: with catecholamine_proxy readings
  }
}

Evidence

  • Vagal nerve stimulation reduces tumor growth in multiple cancer models (DOI 10.3389/fonc.2020.00744)
  • The Shma prayer's sustained exhale IS a parasympathetic hold — "always a vagal protocol"
  • HRV biofeedback improves autonomic balance in pediatric populations
  • Humming generates nitric oxide in paranasal sinuses (Weitzberg & Lundberg, 2002)
Stack: WearShield haptic engine · AudioFabric Enter My Mind breathing sync · watch speaker/haptics · ~0 bytes additional model (rule-based + HRV feedback)
vagal biofeedback zero-cost evidence-based

§3 — Sleep Architecture Regression Predictor

COST: $0 software · watch worn during sleep

Problem

Stage 4S spontaneous regression correlates with healthy autonomic development. Sleep architecture (deep/REM/wake ratios, HRV during sleep, nocturnal autonomic patterns) is a window into the developing nervous system's maturation — which is exactly what regression requires.

Approach

Continuous overnight monitoring builds a sleep maturation profile. A local model compares the child's sleep architecture evolution against trajectories associated with regression vs progression. Early divergence from the regression trajectory triggers clinical review weeks before it would be caught by imaging.

UDT:SleepRegressionPredictor {
  nightly_capture: {
    sleep_stages:     [wake, light, deep, REM] × duration
    hrv_by_stage:     SDNN per stage
    hr_nadir:         lowest HR during deep sleep
    autonomic_shifts: sympathovagal transitions per night
    movement_density: accel events per hour
    temp_nadir:       lowest skin temp (circadian depth)
    respiratory_rate: breaths/min during deep sleep
  }

  maturation_score: {
    deep_sleep_pct:     target > 25% (infant norm)
    hrv_trend:          should be RISING over weeks
    hr_nadir_trend:     should be FALLING (vagal maturity)
    rem_architecture:   should show consolidation

    score = weighted_sum(above) → [0..1]
    regression_trajectory: score > 0.6 and trending UP
    progression_warning:   score < 0.4 or trending DOWN
  }

  model: self-training
    starts with population priors (published infant sleep norms)
    personalizes to this child over 7-14 nights
    no cloud required
}

Evidence

  • Stage 4S regression requires nervous system maturation (PMC5920563)
  • Sleep architecture reflects autonomic nervous system development
  • HRV during sleep is the cleanest window into autonomic state (no movement artifact)
  • Published infant sleep norms provide training priors
Stack: WearShield BackgroundSession (HKObserverQuery overnight) · HealthKit sleep analysis · ~5MB ONNX model · 14-night calibration
sleep predictive non-invasive early-warning

§4 — On-Device Differential Diagnosis Assistant

COST: $0 · WebLLM in browser · no cloud

Problem

In resource-limited settings, pediatric oncology expertise is scarce. A child presents with an abdominal mass — is it neuroblastoma, Wilms tumor, hepatoblastoma, or lymphoma? The differential diagnosis requires pattern recognition that a local health worker may not have.

Approach

A WebLLM model (runs entirely in-browser, no internet required after initial download) fine-tuned on pediatric oncology differentials. Input: age, symptoms, lab values (if available), ultrasound description. Output: ranked differential with next-step recommendations.

UDT:DxAssistant {
  input: {
    age_months:    int
    mass_location: enum [adrenal, renal, hepatic, mediastinal, pelvic, cervical]
    symptoms:      [pain, fever, weight_loss, hypertension, opsoclonus, diarrhea, ...]
    labs: {
      vma_hva:     optional float    // if urine test available
      ldh:         optional float
      ferritin:    optional float
      nse:         optional float    // neuron-specific enolase
    }
    imaging_desc:  free_text         // ultrasound findings
  }

  output: {
    differential: [
      { dx: "neuroblastoma", probability: 0.72, key_features: [...] },
      { dx: "Wilms tumor",   probability: 0.15, key_features: [...] },
      ...
    ]
    next_steps: [
      "obtain urine VMA/HVA (95% sensitivity for NB)",
      "check MYCN amplification status",
      "staging CT chest/abdomen"
    ]
    red_flags: ["opsoclonus-myoclonus suggests paraneoplastic"]
    refer: boolean
  }

  model: Qwen2.5-0.5B or SmolLM-360M
    fine-tuned on: WHO ICCC-3 classification + INRG staging data
    runs: WebLLM in browser · ~300MB download once · no internet after
    language: multilingual (EN/PL/ES/FR/SW/HI)
}

Evidence

  • Age + location + VMA/HVA alone correctly classifies >80% of pediatric abdominal tumors
  • INRG (International Neuroblastoma Risk Group) staging criteria are well-defined decision trees
  • Small LLMs can perform medical differential diagnosis at clinically useful accuracy
Stack: WebLLM (browser) · Qwen2.5-0.5B or SmolLM-360M · fine-tune dataset from INRG + ICCC-3 · offline-capable PWA · multilingual
diagnostic offline multilingual resource-limited

§5 — Post-Immunotherapy Response Tracking

COST: $0 software · existing smartwatch

Problem

Anti-GD2 immunotherapy (dinutuximab) causes significant side effects including pain, fever, and capillary leak. Response monitoring requires periodic imaging. Between scans, there's no continuous signal of whether the treatment is working.

Approach

The immune response to anti-GD2 therapy creates measurable autonomic signatures: inflammation (HRV depression, temperature elevation), pain response (HR spikes), and recovery patterns. A local model tracks these signatures and correlates with treatment response.

UDT:ImmunoTracker {
  treatment_cycle: {
    pre_infusion:    baseline HRV, HR, temp (2h before)
    during_infusion: real-time HRV/HR/temp every 30s
    post_infusion:   recovery trajectory over 24-72h
  }

  biomarkers: {
    inflammation_index:  f(hrv_drop, temp_rise, hr_elevation)
    pain_signature:      f(hr_spikes, movement_reduction, sleep_disruption)
    recovery_slope:      time_to_return_to_baseline
    immune_activation:   hrv_spectral_LF/HF_ratio shift
  }

  response_model: {
    track across cycles (typically 5 cycles):
    if inflammation_index INCREASING cycle-over-cycle → immune engagement ↑
    if recovery_slope SHORTENING → tolerance developing
    if recovery_slope LENGTHENING → cumulative toxicity
    if baseline_hrv IMPROVING between cycles → systemic improvement
  }

  adaptive_support: {
    during pain peaks: auto-trigger vagal protocol (§2)
    pre-infusion: guided breathing to optimize autonomic state
    post-infusion: sleep optimization prompts
  }
}

Evidence

  • Anti-GD2 (dinutuximab) EFS 46%→66% (PMC6306059)
  • Infusion side effects have distinct autonomic signatures
  • HRV recovery kinetics correlate with immune system engagement
  • Pain management through vagal stimulation reduces opioid requirements
Stack: WearShield continuous monitoring · custom ImmunoTracker BioModule · treatment calendar integration · caregiver alerts
immunotherapy monitoring pain-management treatment-response

§6 — Federated Watchnet — Multi-Patient Learning

COST: $0 software · MQTT mesh between consenting families

Problem

Each child's watch collects thousands of data points per day, but a single patient's data is insufficient to train a robust model. Pooling data centrally raises privacy and regulatory concerns.

Approach

Federated learning across a mesh of consenting families' smartwatches. Each watch trains locally on its own data. Only model weight updates (not patient data) are shared via MQTT. The aggregate model improves for everyone while no individual data leaves any device.

UDT:FederatedWatchnet {
  local_training: {
    each watch runs: stochastic gradient descent on local data
    model: shared architecture (tiny LSTM, ~500KB)
    task: predict VMA_proxy from [HR, HRV, temp, sleep, activity]
    epochs_per_round: 5 (on-device, battery-aware)
  }

  aggregation: {
    transport: MQTT (wearshield MQTTClient.swift)
    topic: "watchnet/nb/weights/{round}"
    protocol: FedAvg (McMahan et al. 2017)
    privacy: differential_privacy(epsilon=1.0)
    frequency: 1 round per day (overnight, on charger)
  }

  architecture: {
    watch → local train → weight delta → MQTT → aggregator
    aggregator → averaged weights → MQTT → all watches
    aggregator: can be any family's home server (ASS-OS gateway)
    or: serverless via MQTT broker (no central authority)
  }

  minimum_viable: 5 families × 30 days = 150 patient-days
  improvement_per_family: model accuracy ↑ ~3-5% per additional family
}

Evidence

  • Federated learning preserves privacy while enabling multi-site ML (Google Health, Apple)
  • Neuroblastoma is rare (~700 cases/year US) — federated approach maximizes limited data
  • Smartwatch data volume (100K+ readings/day) compensates for small cohort size
Stack: WearShield MQTTClient · FedAvg aggregation · TensorFlow Lite / ONNX on-device · ASS-OS gateway as optional aggregator · differential privacy
federated privacy multi-patient rare-disease

§7 — SLIT3-Informed Differentiation Biofeedback

COST: $0 software · speculative / research-stage

Problem

SLIT3 drives intratumoral differentiation in Stage 4S (PMC12123822). Can we promote systemic conditions that favor SLIT3 expression or mimic its downstream effects through non-pharmacological means?

Approach

SLIT3-ROBO signaling promotes neuronal differentiation. Differentiation is also promoted by: NGF availability, reduced cortisol, healthy sleep, and vagal tone. A combined biofeedback protocol targets all modifiable factors simultaneously, using the watch to monitor and optimize each.

UDT:DifferentiationOptimizer {
  targets: {
    cortisol_proxy:     minimize via vagal biofeedback (§2)
    sleep_quality:      maximize deep sleep (§3)
    autonomic_balance:  push toward parasympathetic (vagal)
    stress_events:      minimize via caregiver alerts
    activity_level:     maintain age-appropriate movement
    circadian_rhythm:   enforce consistent light/dark cycle
  }

  composite_score: {
    differentiation_favorability =
      (1 - cortisol_proxy) × 0.25 +
      sleep_maturation_score × 0.25 +
      vagal_tone_index × 0.20 +
      circadian_consistency × 0.15 +
      activity_appropriateness × 0.15

    target: > 0.7 (favorable for differentiation)
    track: daily trend over treatment course
  }

  interventions: {
    low cortisol_proxy:    ✓ maintain
    high cortisol_proxy:   trigger vagal session
    poor sleep:            adjust bedtime routine, light exposure
    low vagal tone:        humming exercises, skin-to-skin
    irregular circadian:   light therapy schedule via caregiver app
  }
}

Evidence

  • SLIT3 drives differentiation in 4S (PMC12123822, 2024)
  • Cortisol suppresses differentiation and promotes proliferation
  • Vagal tone correlates with anti-inflammatory, pro-differentiation milieu
  • Sleep quality drives developmental neural maturation in infants
Stack: WearShield BloomState composite score · caregiver app (Aneta site) · vagal protocol integration · light exposure tracking via phone ambient sensor
SLIT3 differentiation speculative multi-modal

§8 — Caregiver AI Companion (Offline)

COST: $0 · WebLLM in browser · works offline

Problem

Parents of children with neuroblastoma face information overload, anxiety, and decision fatigue. Access to pediatric oncology expertise varies dramatically by geography. Questions arise at 3 AM when no one is available.

Approach

The Aneta Dimensional Witch interface, pre-loaded with neuroblastoma-specific knowledge, running entirely in-browser via WebLLM. Available 24/7, offline, in multiple languages. Answers questions about treatment, side effects, monitoring, nutrition, and emotional support. Never replaces clinical advice — always includes "discuss with your care team" with specific questions to ask.

UDT:CaregiverCompanion {
  model: SmolLM-360M or Qwen2.5-0.5B (WebLLM, in-browser)

  knowledge_base: embedded in system prompt
    neuroblastoma staging (INRG)
    treatment protocols (COG standard)
    side effect management
    nutrition for pediatric oncology
    sleep hygiene for hospitalized children
    emotional support frameworks
    when to call the doctor (red flags)

  languages: [EN, PL, JP, ES, FR, HI, SW, AR]

  guardrails: {
    never: diagnose, prescribe, contradict clinical team
    always: end with "discuss with your care team"
    always: generate specific questions for the next appointment
    flag: any mention of self-harm, severe distress → crisis resources
  }

  features: {
    treatment_calendar: track infusion dates, lab draws, imaging
    side_effect_log: structured symptom tracking
    question_bank: accumulate questions for next appointment
    explain_like_5: simplify medical jargon for siblings/family
  }
}

Evidence

  • Caregiver anxiety correlates with worse child outcomes
  • Health literacy interventions improve treatment adherence
  • LLMs perform at medical student level for patient education
  • Offline capability critical for resource-limited and rural settings
Stack: Aneta site (teslasolar.github.io/aneta) · WebLLM · neuroblastoma system prompt · PWA for offline · multilingual
caregiver offline multilingual emotional-support 24/7
